# Защита от читов (кратко)

Экономика server-authoritative — клиент только сообщает события, сервер решает.

- Валидация Telegram initData (HMAC-SHA256) на каждом запросе.
- Энергия считается на сервере (ленивый расчёт) — нельзя начислить монет больше, чем было энергии.
- Мини-игра: серверный seed + nonce, проверка score ≤ теоретического максимума за длительность сессии, кап награды FRUIT_MAX_COINS, отклонение повторного finish.
- Идемпотентные клеймы (daily, upgrade, stake) через ключи в Redis.
- Rate-limit на /tap и /fruit через Redis.
- LedgerEntry — полный аудит движений для разбора аномалий.
